phobos.js

phobos.js allows you to quickly put together a JSON API for use with your front-end applications. It works with familiar, popular open-source modules to give you a rapid head start to building otherwise complex components.

Getting Started

Getting started is simple:

npm install --save phobosjs

After that, phobosjs can be included in your application by using:

var Phobos = require('phobosjs');

What exactly is it?

Phobos.js is a framework built on top of Express.js and Mongoose.js that provides a layer of abstraction for running complex, permissions-driven APIs for Node.js API servers.

Basically, it takes the idea of RESTful resources, mass assignment, finely-grained permissions and bearer token auth and applies it all in a single package, meaning you don't have to. The basic building block of Phobos.js is the controller, something already familiar to you (if you're interested in something like Phobos.js, you've most likely built a few of these).

Why / When would you use it?

You may want to use Phobos.js if:

  • If you need to create a JSON API where you need to clearly lay out RESTful resources and don't want to create a million routes.
  • You are creating an API where Bearer tokens are to be the main authentication mechanism, popularly used in auth frameworks for popular front-end tools like Ember.js, Angular, etc.
  • You want to grant permissions to users to perform reading and CRUD operations on your resources and want to organize these in a neat and concise way.

Most importantly, Phobos.js is here to abstract a lot of the boilerplate code you may have to write to make these things happen. Personally, I built this once for a project, and found that instead of redoing it each time, I wanted to make a module out of it for simplification and maintainability.
